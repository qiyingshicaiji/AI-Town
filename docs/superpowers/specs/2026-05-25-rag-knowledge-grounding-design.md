# RAG Knowledge Grounding — Design Spec

**Date:** 2026-05-25
**Status:** Approved
**Scope:** Sub-project C — prevent NPC hallucination by grounding responses in a knowledge base

## Problem

1. NPCs hallucinate when asked about unknown NPCs or topics they have no experience with
2. NPCs don't know about newly created NPCs (no perceptual memory exists)
3. High temperature (0.9) causes fact fabrication on knowledge questions

## Solution

Three-layer approach:

- **Knowledge RAG**: Markdown knowledge base → chunk → embed → retrieve → inject into prompt
- **Domain Filtering**: Only answer questions within the NPC's expertise; redirect otherwise
- **Dual LLM Call**: Low-temp (0.1) fact extraction → high-temp (0.9) character-styled delivery

## Backend Changes

### 1. `backend/knowledge/ai_town.md` — New File

Single-file knowledge base. `##` headings define retrievable chunks. NPC profiles auto-generated from `NPC_ROLES`.

User writes the static sections (office setting, culture, tutorials). The `## 人物` section is auto-generated by `LoreManager` and overwritten on reload.

### 2. `backend/lore_manager.py` — New File

```
class LoreManager:
    chunks: list[dict]        # [{id, title, content, embedding}]
    _npc_section_template: str

    __init__(knowledge_file: str):
        - Load markdown file
        - If "## 人物" section exists, replace with auto-generated NPC profiles
        - Split by ## headings into chunks
        - Embed each chunk via get_text_embedder()
        - Store embeddings as numpy array

    _generate_npc_section() -> str:
        - Iterate NPC_ROLES
        - For each NPC: "### {name} — {title}\n{personality}\n专长: {expertise}\n"

    query(text: str, expertise_hints: list[str], top_k=3) -> str | None:
        - Embed query text
        - Cosine similarity against all chunks
        - Take top_k chunks
        - Check relevance: if none of the top chunks relate to any expertise hint → return None
        - (Weak filter: if chunk title contains hint keywords, it's relevant)
        - Return joined chunk contents as prompt context

    reload():
        - Re-generate NPC section, re-chunk, re-embed
```

### 3. `backend/scene_generator.py` — Dual Call + Knowledge Injection

New method `_generate_fact(text: str, lore: str) -> str`:
```
- Build prompt: "根据以下参考资料回答问题。只输出事实，不要修饰。\n{参考资料}\n问题: {text}"
- Call SimpleAgent.run(prompt, temperature=0.1)
- Return fact answer
```

Modified `generate_scene()` flow:
```
1. If trigger_message exists:
   a. Extract expertise hints from NPC configs
   b. lore = lore_manager.query(trigger_message, expertise_hints)
   c. If lore:
      - fact = await _generate_fact(trigger_message, lore)
      - Build prompt with fact + NPC roles, temperature=0.9
      - Add rule: "你基于以下事实回答，用你的风格复述"
   d. If no lore:
      - Add rule: "你可能不知道答案，如实说不知道，不要编造"
      - (Optional: redirect to NPC with relevant expertise)
2. Else (no trigger_message, NPC-NPC chat): normal flow, no knowledge injection
```

Domain filtering rule added to prompt when lore exists:
```
【专业约束】
- 如果问题在你的专业领域内，用参考资料准确回答
- 如果超出专业领域且没有相关资料，说"我不太清楚"
```

### 4. `backend/agents.py` — Auto NPC Registration

In `add_npc()`, after creating the NPC:

```python
# Inject perceptual memory into all existing NPCs
for existing_name in self.agents:
    if existing_name != name:
        mem_mgr = self.memories.get(existing_name)
        if mem_mgr:
            mem_mgr.add_memory(
                content=f"办公室来了一位新同事：{name}，职位是{config.get('title','')}。{config.get('personality','')}",
                memory_type="perceptual",
                importance=0.8,
                metadata={
                    "observation_type": "new_colleague",
                    "new_npc": name,
                    "timestamp": datetime.now().isoformat(),
                }
            )

# Reload lore manager to include new NPC in knowledge base
if hasattr(self, 'lore_manager') and self.lore_manager:
    self.lore_manager.reload()
```

### 5. `backend/main.py` — Lore Manager Initialization

Add to lifespan: initialize `LoreManager` on startup, inject into `NPCAgentManager`.
